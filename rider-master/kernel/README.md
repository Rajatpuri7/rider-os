# kernel/ — the operating system

| File | Role |
|---|---|
| `BOOT.md` | Session start sequence + context-loading table. Entry point for every session. |
| `LAWS.md` | 12 inviolable rules. Conflicts escalate to the human. |
| `FAILURE-MODES.md` | Known AI failure patterns by task type + countermeasures. Pre-flight checklist. |
| `PROMPTS.md` | Copy-paste prompts to run RIDER discipline on chat-only models. |
| `META.md` | Rules for changing the kernel; the end-of-project harvest ritual. |
| `CHANGELOG.md` | Kernel version history. |
| `protocols/` | Numbered step-by-step procedures for every recurring cognitive activity. |
| `templates/` | Fill-in skeletons for every artifact type (task, ADR, test report...). |
| `playbooks/` | Domain expertise packs (embedded, CV, ML, controls...). Loaded per PROJECT_PROFILE domain flags. |

Modification rules: see `META.md`. Default answer to "should I edit the kernel
mid-project": no — log a lesson instead.
