# QUESTIONS — the map of ignorance

> Open questions that shape design or block work. A LONG list is healthy —
> it means unknowns are visible instead of ambient. Review at every audit (P09).
> Research sessions (P07) start by picking one entry here.

Format per entry:

```
## Q-### — <the question, sharply>
- Status: open | researching | answered | obsolete
- Raised: YYYY-MM-DD, by <session/task>
- Blocks: <task/decision it gates, or "shapes design broadly">
- What an answer looks like: <so research knows when to stop>
- Answer (when resolved): <one line + link to research note / experiment / human word>
```
